numba.cuda is a Python library for writing and executing CUDA kernels directly from Python.
It provides a NumPy-like programming model for GPU computation, allowing users to write
CUDA kernels using Python syntax and launch them on NVIDIA GPUs via JIT compilation.

Below is a simple guide to the library and its core concepts.

Getting Started
This guide introduces the basic concepts of numba.cuda and how to use it for GPU programming.

CUDA Programming Model
numba.cuda exposes a low-level CUDA programming model similar to CUDA C, including
thread indexing, grid/block configuration, shared memory, and device synchronization.

Kernels are defined as Python functions decorated with @cuda.jit and executed on the GPU.

import numba.cuda as cuda
import numpy as np

@cuda.jit
def add_kernel(a, b, out):
    i = cuda.grid(1)
    if i < out.size:
        out[i] = a[i] + b[i]

a = np.arange(10, dtype=np.float32)
b = np.arange(10, dtype=np.float32)
out = np.zeros_like(a)

threads_per_block = 128
blocks = (a.size + threads_per_block - 1) // threads_per_block
add_kernel[blocks, threads_per_block](a, b, out)

Device Arrays
numba.cuda provides device arrays that live in GPU memory.
These arrays can be explicitly allocated, transferred between host and device,
and used inside CUDA kernels.

import numba.cuda as cuda
import numpy as np

a = np.arange(5, dtype=np.float32)
d_a = cuda.to_device(a)

# Allocate an output array directly on the device
d_out = cuda.device_array_like(a)

@cuda.jit
def square_kernel(x, out):
    i = cuda.grid(1)
    if i < out.size:
        out[i] = x[i] * x[i]

square_kernel[1, 32](d_a, d_out)
result = d_out.copy_to_host()
print(result)

Thread Indexing and Memory Access
numba.cuda exposes CUDA thread and block indices via utility functions.
These allow kernels to determine which element of the data each thread operates on.

Common indexing helpers include:
- cuda.threadIdx.x
- cuda.blockIdx.x
- cuda.blockDim.x
- cuda.grid(1)

import numba.cuda as cuda

@cuda.jit
def scale_kernel(x, factor):
    i = cuda.grid(1)
    if i < x.size:
        x[i] *= factor

Synchronization and Shared Memory
numba.cuda supports shared memory and thread synchronization within a block.
Shared memory can be declared using cuda.shared.array, and threads can be synchronized
using cuda.syncthreads().

import numba.cuda as cuda
from numba import float32

@cuda.jit
def sum_block_kernel(x, out):
    sm = cuda.shared.array(128, dtype=float32)
    tid = cuda.threadIdx.x
    i = cuda.grid(1)

    if i < x.size:
        sm[tid] = x[i]
    else:
        sm[tid] = 0.0

    cuda.syncthreads()

    if tid == 0:
        s = 0.0
        for j in range(cuda.blockDim.x):
            s += sm[j]
        out[cuda.blockIdx.x] = s

Writing GPU-Oriented Code
numba.cuda kernels must be written with explicit parallelism and memory behavior in mind.
Unlike NumPy-style vectorized code, kernels typically operate on individual elements
and rely on grid and block configuration for parallel execution.

When writing solutions, prefer using:

    import numba.cuda as cuda

and call APIs via the `cuda` namespace.
