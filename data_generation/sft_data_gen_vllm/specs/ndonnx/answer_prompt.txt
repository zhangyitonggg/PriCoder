{

}

### answer requirements
- Must include: `{required_import}`
- Must define EXACTLY ONE top-level function; put all logic inside that function
- The function MUST have a docstring and include an "Examples" section (or doctest-style `>>>`)
- Follow Python style; include necessary comments and type hints
- Avoid randomness; avoid network/file I/O; prefer small deterministic inputs

### test requirements
- Must contain at least {min_asserts} `assert ...` statements
- Call the target function multiple times and cover normal + edge cases
- Must be deterministic (no randomness/time/external environment)

- It is not necessary to use all provided seed APIs; avoid forcing API usage when it does not meaningfully contribute to solving the task.
- Do NOT invent APIs that do not exist (e.g., ndonnx is not the same as numpy).
